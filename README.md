# modarn-css
sassに近いcssの記法や、運用を支える背景情報などを踏まえたプロパティを紹介する

## sample01 スタイルの記述場所について

headの外で書かれたスタイルの動作を検証する。  

- 詳細度が同じ場合は後で書いたもので上書きされるルールは同じ。  
- head内に書くことが「推奨」されている
- レイアウトが大きく変わるようなcssが後で読まれる場合はSEOに悪影響を与える可能性がある
- CMSなどで特有のページにだけスタイルを当てたい場合に重宝する

## sample02 pictureによる読み込み画像の切り替えについて

picture要素によって細かく制御できる画像の読み込みについて。  
スタイルを当てる時は、pictureではなくimgとして記述する。  

- 1つのタグでレスポンシブ化やretina対応が記述できる
- webpなど、古いブラウザ対応が必要な場合もまとめて仕分けができる
- pictureタグは上から順番に評価し、どれも当てはまらない場合は最後のimgタグの内容が表示される

## sample03 imgのloading属性について

imgに「loading="lazy"」を記述することによって、画像の遅延読み込みが可能になる。  
目視での確認は難しいため、「開発者ツール」 => 「ネットワーク」 を開いた状態でリロードし、スクロールしながら読み込みの状況を確認しよう。

### 遅延読み込みのメリット

- 初期アクセスで読み込むリソース量が減るため初期表示が早くなる
- コンテンツロードも早くなるため画像のロードを待たずスクリプトを実行できる

### 使用の注意点

- KVなど最初に読むべきリソースで使うとかえって読み込み時間が長くなる
- widthとheightの併用が必要(あらかじめエリアを確保しておくため: レイアウトシフト対策)
- onloadの方が先に発火するため、初期の画像サイズを前提としたスクリプトとの相性が悪い

## sample04 変数の利用

cssでもsassのように変数が使用可能となっている。  
sassに比べると書き方に癖が感じられるかもしれないが大きく使い方は変わらない。  

### sassの変数との違い

- 変数の宣言に「--」を接頭語として付けること
- 呼び出しにはvar(変数名)を使い、プロパティの値としてのみ使うこと
- メディアクエリには使えないこと

誤った記述の例として「media.html」を用意している。ブラウザで開発者ツールを開いて動作を見てみよう。


## sample05 cssでのネストについて

cssでネスト(入れ子)にして階層でクラスを当てることができる。  
詳細度は高くなっていくため、階層が深くなりすぎないように注意。  
また、sassで使えたような&による自身までのクラスを省略するような表記も可能。  

## sample06 サブGridレイアウト

display: gridでは親子関係によるグリッドを作成することができる。  
例えば、カード型要素のようにテキストの長さが可変の場合でも、親要素の高さを合わせたい時などに重宝する。  
記述が複雑になりがちなため、端の揃ったデザインにつられて無駄に使わないように注意しよう。

## sample07 グリッドエリアによるシンプルなHTMLのレスポンシブレイアウト

display: gridを使った場合、display: flexのorderのようにHTMLの記述順とは別の表示順を与えることができる。  
flexに比べるとエリアサイズの指定などレイアウトに向いている。  
可変数のコンテンツにはflexの方が使いやすい。  

## sample08 画像レイアウト

object-fitやaspect-ratioを使った比率を担保したデザイン構築。  
background-imageでほぼ同じことができる。  

### imgを使う場合の利点

- 横や高さを細かく指定しなくても使いやすい
- 非同期読み込みに対応できる
- アクセシビリティ対応やSEO対策などに強い

## sample09 コンテンツのセンタリング

flexやgridなどを使わずコンテンツに合わせて上下左右指定を可能とする方法。  
positionを使う必要がないためデザインを崩しにくい。  
複雑な構造の場合はflexなどを使い続けた方がいいこともあるためコンテンツ量が多い時は避けることも検討しよう。  

## sample10 切り抜きの指定

cssを使った要素の切り抜きを行う。  
三角形などのサンプルは昔からあるが、ポリゴン指定によってより複雑な表現が可能となった。  
「css polygon」で検索するか、AIに作ってもらうかしよう(覚えてもいいが割に合わないと思われる)。  

## sample11 レスポンシブなプロパティ指定

cssにもsassのような関数が存在し、柔軟な指定が可能となる。  
特に「clamp」はレスポンシブサイトに強く、デザイン通りのコーディングをするにはかなり有効。  
ブレークポイント付近に気をつければ、フォントサイズやマージンまで割合指定できるためデザインを伸縮したような表現ができる。
